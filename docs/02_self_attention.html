📖 Notebook 02: Self-Attention（自己注意機構）¶

— Transformer の心臓部を数値で追う —¶

なぜ Attention が必要か¶

「銀行」 という単語の意味は文脈によって変わります：

  • 「川の 銀行 に立つ」→ 川岸
  • 「お金を 銀行 に預ける」→ 金融機関

RNN（前世代のモデル）は左から右に処理するため、遠い文脈を参照しにくいという問題がありました。 Self-Attention は全トークンを並列に・直接参照できます。

計算の全体像¶

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$

記号 名前 役割 検索エンジンの比喩
Q Query（クエリ） 何を探しているか 検索ワード
K Key（キー） どんな情報を持つか 文書のタグ
V Value（バリュー） 参照されたときに渡す情報 文書の中身

    return PE

Step 1: Q・K・V 行列の生成¶

入力 $X$ に 3 種類の重み行列を掛けて Q/K/V を生成します：

$$Q = X W_Q \in \mathbb{R}^{L \times d_k}$$ $$K = X W_K \in \mathbb{R}^{L \times d_k}$$ $$V = X W_V \in \mathbb{R}^{L \times d_v}$$

  • $W_Q, W_K, W_V \in \mathbb{R}^{d_{\text{model}} \times d_k}$ は学習パラメータ
  • ここでは $d_k = d_{\text{model}} = 4$

直感：同じ X から 3 種類の「見せ方」を作る。 Q は「質問」、K は「索引」、V は「回答」の視点から X を変換します。

Step 2: Attention スコアの計算¶

$$\text{scores} = \frac{QK^\top}{\sqrt{d_k}} \in \mathbb{R}^{L \times L}$$

scores[i, j] = 位置 i のトークンが位置 j のトークンを「どれだけ参照したいか」

なぜ $\sqrt{d_k}$ で割るか：

  • $d_k$ が大きいほど内積の値も大きくなり、softmax が極端な分布になる
  • $\sqrt{d_k}$ で割ることで分散を 1 付近に正規化する

証明：$x, y \sim \mathcal{N}(0,1)$ のとき $x \cdot y \sim \mathcal{N}(0, d_k)$
→ $\sqrt{d_k}$ で割れば $\mathcal{N}(0,1)$ に戻る

Step 3: Causal Mask（因果マスク）¶

GPT 系の デコーダ は「未来のトークンを参照してはいけない」ルールがあります。 （エンコーダは双方向なのでマスク不要）

なぜか： テキスト生成時は未来のトークンが存在しない。 学習時にもその条件を揃えるため、上三角をマスクします。

$$\text{scores\_masked}[i,j] = \begin{cases} \text{scores}[i,j] & (j \leq i) \\ -\infty & (j > i) \end{cases}$$

$-\infty$ を softmax に入れると $e^{-\infty} = 0$ になり、参照確率が 0 になります。

Step 4: Softmax → Attention 重み¶

各行（各クエリトークン）に独立に softmax を適用します：

$$\text{attn\_weights}[i,j] = \frac{e^{\text{scores}[i,j]}}{\sum_{k} e^{\text{scores}[i,k]}}$$

結果：

  • 各行の合計 = 1（確率分布）
  • マスク位置（-1e9）は $e^{-10^9} \approx 0$ となり実質 0

数値安定化： $e^{\text{large}}$ はオーバーフローする恐れがあるため、 最大値を引いてからべき乗します（値は変わらない）：

$$\text{softmax}(z_j) = \frac{e^{z_j - \max z}}{\sum_k e^{z_k - \max z}}$$

def softmax(x, axis=-1):
    e = np.exp(x - x.max(axis=axis, keepdims=True))
    return e / e.sum(axis=axis, keepdims=True)

Step 5: V との重み付き和 → Attention 出力¶

$$\text{output} = \text{attn\_weights} \cdot V \in \mathbb{R}^{L \times d_v}$$

各トークンの出力ベクトル = 参照したトークンの Value ベクトルの加重平均です。

例: 位置 2 の出力 = 0.47×V[0] + 0.29×V[1] + 0.24×V[2] + 0.00×V[3]

# 完全な Scaled Dot-Product Attention 関数
def scaled_dot_product_attention(Q, K, V, causal=True):
    """
    Vaswani et al. (2017) Eq.(1)
    Returns: output (L, d_v), weights (L, L)
    """
    seq_len, d_k = Q.shape
    scores = Q @ K.T / np.sqrt(d_k)
    if causal:
        mask = np.triu(np.ones((seq_len, seq_len), dtype=bool), k=1)
        scores[mask] = -1e9
    weights = softmax(scores, axis=-1)
    return weights @ V, weights

Multi-Head Attention（多頭注意機構）¶

1 つの Attention ヘッドでは 1 種類の「参照パターン」しか学べません。

Multi-Head Attention は $h$ 個の独立したヘッドを並列に動かします：

$$\text{MultiHead}(X) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\, W^O$$

$$\text{head}_i = \text{Attention}(XW^Q_i,\; XW^K_i,\; XW^V_i)$$

  • 各ヘッドの次元 $d_k = d_{\text{model}} / h$
  • ここでは $h = 2$、$d_k = 4/2 = 2$

各ヘッドが異なるパターンを学習：

  • ヘッド 1：文法的な依存関係
  • ヘッド 2：意味的な類似性
    など（実際には解釈は難しいですが）

まとめ：Attention の計算フロー¶

X (4, 4)
 │
 ├──@W_Q──→ Q (4,4) ─┐
 ├──@W_K──→ K (4,4) ─┼─→ Q@K^T/√d_k → softmax+mask → weights (4,4) ─→ @V → head_out (4,4)
 └──@W_V──→ V (4,4) ─┘
       ↑ これを num_heads 回繰り返す
 Concat → (4, d_model) → @W_O → MHA出力 (4, 4)
操作 入力 shape 出力 shape 意味
X @ W_Q (4,4)×(4,4) (4,4) Q の生成
Q @ K.T (4,4)×(4,4) (4,4) 全ペアのスコア
/ √d_k (4,4) (4,4) 正規化
softmax (4,4) (4,4) 確率化
weights @ V (4,4)×(4,4) (4,4) 重み付き和
Concat+@W_O (4,4) (4,4) Multi-Head の統合
